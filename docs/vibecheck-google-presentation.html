Demo Day Framing

Most swipe apps optimize for heat, not fit.

Shallow Discovery

Traditional swipe apps over-index on a photo-first decision and under-weight communication style, taste, and intent.

Weak Match Quality

Users get plenty of activity but too little signal about who they will actually enjoy talking to after the first hello.

No Real Memory

Without persistent context, product decisions do not improve over time and matches feel disposable instead of compounding.

VibeCheck reframes the product around behavioral intelligence: first understand the user’s tone, archetype, and preferences, then use those signals to improve discovery, messaging, and match prioritization.

Architecture

Modern web app with an AI decision layer.

Frontend
  • React 19 + TypeScript + Vite
  • Tinder/Bumble-style discover deck with gallery viewer
  • Persistent match inbox with timestamps, unread state, and simulated incoming replies
  • Simulation UI that runs a vibe check before starting the AI conversation
Backend
  • FastAPI + SQLAlchemy + SQLite today
  • Profile storage, photo upload handling, message history, and match-thread persistence
  • AI endpoints for vibe profiling, simulation completion, and reply generation
  • Railway deployment path with env-driven Gemini configuration

The product is already structured so the AI layer is not decorative. It directly affects matching, inbox behavior, and how the deck is prioritized.

Google-Native Roadmap

What a deeper Google stack could unlock next.

Firebase Auth

Replace local auth handling with a stronger identity layer and easier provider sign-in for consumer onboarding.

Cloud Firestore

Turn match threads into real-time documents instead of app-local JSON blobs, making the inbox truly multi-user.

Cloud Storage

Move profile gallery assets into Google-managed storage with cleaner delivery and scaling.

Analytics + App Check

Measure onboarding drop-off, match quality, and protect abuse-sensitive AI endpoints at consumer scale.

The important distinction for Demo Day: Gemini is already live in the product. Firebase is best described as scaffolded and strategically plausible, not yet core to runtime.
